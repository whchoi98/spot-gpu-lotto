# GPU Spot Lotto — Plan 5: Helm & K8s

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Create a Helm umbrella chart for deploying all 4 GPU Spot Lotto services (api-server, dispatcher, price-watcher, frontend) to the Seoul EKS cluster, plus K8s manifests for Karpenter NodePool, storage PVs, NetworkPolicies, and ServiceMonitors.

**Architecture:** Single Helm chart `helm/gpu-lotto/` with sub-templates per service. Each service gets its own Deployment, ServiceAccount, and optional Service/Ingress/HPA/PDB. Values files separate dev/prod config. Existing `k8s/` manifests for Karpenter and storage are already in place.

**Tech Stack:** Helm 3, Kubernetes 1.31, AWS ALB Ingress Controller, Karpenter, Prometheus ServiceMonitor CRD

**Spec:** `docs/superpowers/specs/2026-04-03-gpu-spot-lotto-design.md` (sections 8, 9.6, 10)

**Depends on:** Plan 1 (Python Backend), Plan 3 (Frontend), Plan 4 (Terraform)
